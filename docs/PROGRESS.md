## Как запускать / тестировать (заполнять по мере появления кода)

- Установка: `pip install -e ".[dev]"` (Python 3.11+).
- **Окружение:** пакет `arena` установлен editable в `.venv` репозитория. Запускать
  тесты/код именно через него: `\.venv\Scripts\python.exe -m pytest`
  (системный `python` пакет `arena` не видит → `ModuleNotFoundError: No module named 'arena'`).
- Тесты: `\.venv\Scripts\python.exe -m pytest` (сейчас 559 passed, 1 skipped: +★ Phase 8 — engine cache + stats aggregate + stats report/combined PGN + tournament pairings + tournament runner — и CLI (models/play/tournament); config + catalog + board + endgame + move parsing + models + pgn + pgn export + providers base/openai/anthropic/gemini/transport/retry + obs logging + web logging + engine factory + web engine + full e2e (база+★) + arena player + arena runner (вкл. протокол подсказок ★) + prompts system + prompts context (+ fixtures) + storage game store (+ pgn export + pgn opens as valid game) + report board image (PNG skip без cairosvg) + report html template (вкл. сводку ★-анализа + интерактивный плеер) + report render from fixture + engine stockfish (real-binary тест проходит — движок в `tools/bin`) + analysis analyzer ★ + analysis classify ★ + analysis commentary ★ + web app skeleton ★ + web model selection ★ + web start game ★ + web live view ★ + web games view ★ + config provider names + optional temperature (omit when null) + arena e2e + smoke; единственный skip — PNG-рендер без `cairosvg`).
- Запуск веб-UI: `\.venv\Scripts\python.exe -m uvicorn arena.web.app:app` (выбор моделей `/games/new`, старт `POST /games`, живой просмотр, список/отчёты `/games`).
- Пример партии: `\.venv\Scripts\python.exe scripts\generate_sample_game.py` → `examples/sample-game/` (game.json+PGN+report.html, реальный анализ Stockfish).
- **CLI** (`src/arena/cli.py`): `\.venv\Scripts\python.exe -m arena.cli models` (каталог +
  наличие ключей), `… play WHITE BLACK [--id … --max-plies N --no-persist]` (одна партия →
  `games/<id>/`), `… tournament M1 M2 [M3 …] [--double --id …]` (round-robin → таблица +
  `games/tournaments/<id>/`). Вывод переведён на UTF-8 (кириллица/значки на Windows).

## Открытые хвосты / заметки

- Stockfish (опциональная зависимость) **установлен и проверен** на машине разработки:
  пребилт Stockfish 18 (bmi2) лежит в `tools/bin/stockfish.exe` (в git не коммитится,
  `.gitignore`). Корневой `conftest.py` добавляет `tools/bin` в `PATH` только на время
  тестов, поэтому интеграционный тест движка теперь **выполняется** (не skip). Для
  прогона приложения вне pytest движок берётся из PATH или `engine.path` в `config.yaml`
  (provisioning PATH вне тестов — на усмотрение пользователя; систему не меняли).
- GitHub-remote не подключён (не требуется для локальной работы; можно добавить позже).
