# PROJECT SPEC — LLM Chess Arena

## 1. Цель

Платформа, где две LLM-модели от разных провайдеров играют партию в классические
шахматы без контроля времени. Система выступает арбитром: проверяет легальность
ходов, ведёт полный лог, формирует артефакты партии (PGN + HTML-отчёт).

## 2. Поддерживаемые провайдеры

| Провайдер | SDK            | Пример моделей                  |
|-----------|----------------|---------------------------------|
| OpenAI    | `openai`       | gpt-4o, o-series                |
| Anthropic | `anthropic`    | claude-opus, claude-sonnet      |
| Google    | `google-genai` | gemini-2.x                      |

Каталог конкретных моделей задаётся в `config.yaml` и может расширяться без правок кода.

## 3. Функциональные требования

### 3.1 Конфигурация
- API-ключи и настройки провайдеров вынесены из кода.
- Секреты — в `.env`; несекретные настройки (каталог моделей, путь к движку,
  лимиты) — в `config.yaml`.

### 3.2 Выбор моделей
- Пользователь через **веб-интерфейс** выбирает две модели (белые/чёрные) из каталога.
- Можно выбрать одинаковую модель с обеих сторон (зеркало) — разрешено.

### 3.3 Игра
- Классические правила, без контроля времени.
- Ход делает модель, чья очередь; FEN/PGN ведутся системой.
- Окончание: мат, пат, недостаточный материал, троекратное/пятикратное повторение,
  правило 50/75 ходов, техническое поражение (см. 3.4), сдача (если модель её заявит).

### 3.4 Валидация ходов и технический проигрыш
- Каждый ход проверяется на легальность через `python-chess`.
- Если ход нелегален или не распознан — система отправляет модели корректирующее
  сообщение (причина + список легальных ходов) и просит сыграть заново.
- **Три нелегальные попытки подряд на одном ходу → техническое поражение** этой модели.
- Счётчик попыток сбрасывается после успешного легального хода.

### 3.5 Логирование (что обязательно сохраняем)
- Все ходы партии (SAN + UCI + FEN до/после).
- Полную историю сообщений с каждой моделью (system / context / ответы / коррекции).
- Рассуждение/объяснение модели для каждого хода.
- Все нелегальные попытки с причиной отклонения.
- Использование подсказок, оценки движка, итоговую классификацию ходов.

### 3.6 Информация, передаваемая модели на каждом ходу
- Цвет модели и номер хода.
- **FEN** текущей позиции.
- **Полный список легальных ходов** (в SAN).
- **PGN** партии на текущий момент.
- Предыдущие ходы.
- Предыдущие объяснения **обеих** моделей.
- Сколько подсказок осталось.
- При повторном запросе — причина отклонения предыдущего хода.

### 3.7 Выходные артефакты
- **PGN-файл**, корректно открывающийся в chess.com и lichess (стандартные теги +
  ходы SAN; рассуждения и аннотации — в комментариях `{...}` и NAG).
- **HTML-отчёт** (self-contained): картинки доски по ходам, ходы, объяснения моделей,
  классификация ходов, оценки движка, итог партии.

## 4. ★ Дополнительные требования

### 4.1 Подсказки от движка
- У каждой модели **3 подсказки на партию**.
- Модель может запросить подсказку в своём ответе (поле протокола `request_hint`).
- Stockfish возвращает лучший ход и краткую оценку текущей позиции.
- Запрос уменьшает счётчик; факт и содержание подсказки логируются.

### 4.2 Авторазметка интересных ходов
- После партии каждый ход оценивается Stockfish (centipawn loss относительно
  лучшего хода с точки зрения ходившей стороны).
- Классы (пороги конфигурируемы): `blunder` (зевок), `mistake` (ошибка),
  `inaccuracy` (неточность), `good`, `brilliant` (блестящий — эвристика: лучший
  ход + жертва материала / единственный спасающий).
- Опционально: топ-линии движка + рассуждение модели передаются LLM для
  естественно-языкового комментария и подтверждения «блестящего» хода.

## 5. Форматы данных

### 5.1 `config.yaml` (несекретный)
```yaml
arena:
  illegal_move_retries: 3        # попыток до технического поражения
  hints_per_player: 3
  auto_claim_draws: true         # авто-ничья при 50-ходовом/троекратном повторении
engine:
  enabled: true
  path: "stockfish"              # путь к бинарнику или имя в PATH
  analysis_depth: 18
  hint_depth: 18
providers:
  openai:    { api_key_env: OPENAI_API_KEY }
  anthropic: { api_key_env: ANTHROPIC_API_KEY }
  google:    { api_key_env: GOOGLE_API_KEY }
models:                          # каталог выбираемых моделей
  - id: gpt-4o
    provider: openai
    display_name: "GPT-4o"
    params: { temperature: 0.2, max_tokens: 1024 }
  - id: claude-opus-4-8
    provider: anthropic
    display_name: "Claude Opus 4.8"
    params: { temperature: 0.2, max_tokens: 1024 }
  - id: gemini-2.5-pro
    provider: google
    display_name: "Gemini 2.5 Pro"
    params: { temperature: 0.2, max_tokens: 1024 }
output:
  games_dir: "games"
```

### 5.2 `.env` (секреты, gitignore)
```
OPENAI_API_KEY=...
ANTHROPIC_API_KEY=...
GOOGLE_API_KEY=...
```
Коммитится только `.env.example` с именами переменных без значений.

### 5.4 Структура папки партии
```
games/<id>/
├── game.json        # источник истины
├── game.pgn         # экспорт для lichess/chess.com
├── report.html      # self-contained отчёт
└── boards/          # картинки доски по полуходам (опц., иначе inline в HTML)
```

## 6. Нефункциональные требования
- Ключи не утекают в артефакты/логи (маскирование).
- Устойчивость к сбоям провайдера: ретраи с backoff, понятные ошибки.
- Stockfish — опциональная зависимость: без него работают база (3.1–3.7), ★-фичи
  деградируют корректно (отключаются с предупреждением).
- Воспроизводимость: партия полностью реконструируется из `game.json`.

## 7. Вне области (на данный момент)
- Турниры/рейтинги/лиги нескольких партий.
- Контроль времени, блиц.
- Дообучение моделей, собственный шахматный ИИ.
