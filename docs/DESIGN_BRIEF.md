# DESIGN BRIEF — LLM Chess Arena

Документ для генерации **всего дизайна сайта** (вход для дизайн-инструмента/функции
`design`). Описывает продукт, визуальное направление, все экраны, компоненты,
состояния и потоки. Включает как **существующие** функции, так и **новые**
(помечены 🆕). Цель: по этому брифу можно нарисовать каждый экран в обеих темах.

---

## 1. О продукте

**LLM Chess Arena** — арена, где две LLM-модели (OpenAI, Anthropic, Google Gemini
и др.) играют друг против друга в классические шахматы без контроля времени.
Система проверяет легальность каждого хода, ведёт полный лог (ходы, рассуждения,
ошибки), на выходе даёт PGN и интерактивный отчёт с ★-анализом качества ходов
(оценки в стиле chess.com), подсказками движка (Stockfish) и приватной «стратегией»
каждой модели. Есть одиночные партии и турниры (round-robin) с таблицей.

Аудитория: интересующиеся ИИ и шахматами; разработчики; зрители «битвы моделей».
Партии = реальные платные вызовы API → в UI всегда видны предупреждения о расходах.

## 2. Визуальное направление

- **Стиль:** тёплый «деревянный» шахматный — палитра дерева/крема/зелёного сукна,
  классический «турнирный зал», но современная вёрстка (скругления, мягкие тени,
  аккуратная сетка и типографика).
- **Темы:** 🆕 **светлая и тёмная** с переключателем (по умолчанию — системная,
  выбор сохраняется). Обе темы обязательны для каждого экрана.
- **Палитра (ориентир):**
  - Светлая: фон-пергамент `#efe5d2`, карточки `#fbf5ea`, текст `#3b2f23`,
    приглушённый `#8a7a64`, линии `#e3d4ba`, акцент-сукно `#5f7d36`, дерево `#7a5733`.
  - Тёмная: фон `#1e1812`, карточки `#2a2118`, текст `#ece2d2`, приглушённый
    `#b3a085`, линии `#3c3024`, акцент `#8aab53`, дерево `#2c2016`.
- **Акцентный цвет** — зелёное сукно (основные кнопки, активные состояния).
- **Типографика:** для заголовков/бренда — благородный serif (классика шахмат);
  для текста и данных — гротеск/system-ui; цифры таблиц — табличные (моноширинные).
- **Иконография:** шахматные глифы, минималистичные иконки. Доска — узнаваемая,
  тёплых тонов, с зелёной подсветкой последнего хода.

---

## 3. Глобальные элементы (на всех страницах)

- **Шапка (header):** бренд-логотип «♟ LLM Chess Arena» (ссылка на главную) +
  основная навигация + переключатель темы 🌙/☀️ + 🆕 переключатель языка RU/EN +
  🆕 меню пользователя (аватар/логин).
- 🆕 **Навигация (nav):** постоянное меню — Главная · Сыграть · Архив · Турниры ·
  (Войти/Профиль). Активный пункт подсвечен. Должно работать как набор
  deep-link-страниц: любой URL открывается напрямую и переживает перезагрузку.
- **Подвал (footer):** копирайт, ссылка на GitHub/доку, краткий дисклеймер про
  платные вызовы API.
- **Глобальные состояния для отрисовки:** загрузка (скелетоны/спиннер), пустое
  состояние (нет партий/моделей), ошибка (баннер), тост-уведомления.

---

## 4. Языки 🆕

- При первом заходе — **выбор языка RU/EN** (приветственный экран/модалка).
- После выбора **весь интерфейс только на выбранном языке** (EN-only или RU-only),
  без смешения. Переключатель языка доступен в шапке в любой момент.
- Выбор языка сохраняется (cookie/профиль) и переживает перезагрузку.
- 🆕 **Язык ответов моделей** следует выбору интерфейса: рассуждения/комментарии
  моделей просим выдавать на выбранном языке.
- Для дизайна: показать (а) экран/модалку выбора языка, (б) переключатель в шапке,
  (в) что тексты длиннее в RU — резервировать место (кнопки/таблицы не должны ломаться).

---

## 5. Модели и провайдеры 🆕

- Каталог моделей по провайдерам (OpenAI, Anthropic, Google Gemini; расширяемо).
- 🆕 **Больше моделей** и для каждой — **выбор режима: «думающая» (thinking/
  reasoning) или «быстрая» (flash)**. Думающая — глубже/дороже/медленнее; flash —
  дешевле/быстрее/стабильнее.
- Для каждой модели в выборе показывать: провайдер, название, **бейдж режима**
  (think/flash), подсказку по стоимости/скорости, статус ключа (доступна/«ключ не
  задан» → задизейблена).
- Группировка по провайдерам; поиск/фильтр при большом списке.
- Для дизайна: карточка модели в селекторе (иконка провайдера, имя, бейдж
  think/flash, инфо-иконка с тултипом «дороже/медленнее»), переключатель режима.

## 6. Аккаунты и регистрация 🆕

- 🆕 **Регистрация / вход пользователя.** Экраны: регистрация, вход,
  восстановление пароля (опц.), профиль.
- **Меню пользователя** в шапке: аватар/имя → Профиль · Мои партии · Выход.
  Для гостя — кнопки «Войти» / «Регистрация».
- **Профиль:** имя, e-mail, аватар, настройки (язык, тема), 🆕 свои API-ключи (опц.,
  чтобы играть на своих ключах), история своих партий и турниров.
- **Мои партии:** список партий, созданных пользователем (отдельно от общего архива).
- Состояния форм: валидация полей, ошибки («e-mail занят», «неверный пароль»),
  успех, загрузка.
- Для дизайна: формы регистрации/входа (в тёплой деревянной эстетике), дроп-меню
  пользователя, страница профиля с вкладками.

## 7. Экраны (страницы)

### 7.1. Главная (`/`)
- Герой-блок: название, краткое описание продукта, крупные CTA-кнопки:
  «Сыграть партию», «Архив партий», «Турнир», «Турниры».
- Опц. витрина: примеры партий/превью доски, ключевые фичи (★-анализ, подсказки,
  стратегия), дисклеймер про платные вызовы.
- Состояния: обычное; для гостя — приглашение зарегистрироваться.

### 7.2. Новая партия (`/games/new`)
- Форма выбора **белых** и **чёрных**: два селектора моделей (см. §5) + 🆕 выбор
  режима think/flash для каждой стороны.
- Параметры партии (опц., из конфига): лимит ходов, подсказки вкл/выкл, стратегия
  вкл/выкл.
- Кнопка «Начать». Предупреждение о расходах. Ошибки (нет ключа, не выбрана модель).
- Для дизайна: две колонки «Белые/Чёрные» с карточками моделей, акцент-кнопка старта.

### 7.3. Живая партия (`/games/{id}`, идёт)
- **Двухколоночный layout:** слева доска (тёплая, зелёная подсветка последнего хода,
  плавная анимация скольжения фигур), справа два блока:
  1. **Ходы** — парами (белый-чёрный), текущий ход подсвечен;
  2. **Мысли текущей модели** — заголовок «N. ход · модель» + рассуждение.
- Статус-строка («Партия идёт…», «Завершено…»). Real-time по WebSocket.
- 🆕 **Управление партией:** кнопки **Пауза / Продолжить**; индикатор «на паузе».
  На паузе ходы не запрашиваются; продолжение возобновляет с того же места.
- 🆕 **Устойчивость к перезагрузке:** перезагрузка страницы восстанавливает текущее
  состояние партии (переподключение WS, дозагрузка сыгранных ходов), не «умирает».
- По завершении — авто-переход к отчёту с анализом.
- Состояния: идёт / на паузе / завершается / ошибка провайдера / переподключение.

### 7.4. Отчёт о партии (`/games/{id}`, завершена)
- Самодостаточный интерактивный плеер (также скачивается как offline-HTML):
  - Одна доска + **навигация**: ◀ / ▶ + слайдер + клик по ходу + клавиши ←/→
    (кнопки «в начало/конец» убраны). Анимация скольжения хода (≈ chess.com).
  - **Список ходов парами**, каждый ход **раскрашен по оценке** (см. §8).
  - Панель текущего хода: подписанные блоки **«Мысли модели»** и **«План»**,
    бейдж классификации + оценка (cp), подсказка движка (если была).
  - Блок **«Анализ партии»**: точность по сторонам, счётчики
    зевков/ошибок/неточностей.
  - Кнопки: **Скачать PGN**, **На главную**.
- Для дизайна: показать плеер с раскрашенными оценками в обеих темах.

### 7.5. Архив партий (`/games`)
- Список партий (карточки/строки): игроки и модели, итог (1-0/0-1/½-½/идёт),
  статус-бейдж (идёт/на паузе 🆕/завершена/ошибка), id, дата.
- 🆕 Фильтры/поиск; для авторизованного — переключатель «Все / Мои».
- Состояния: список / пусто / загрузка.

### 7.6. Новый турнир (`/tournaments/new`)
- Выбор ≥2 моделей (чекбоксы/карточки, с режимом think/flash 🆕), флаг «два круга»
  (double round-robin). Валидация (≥2 модели, ключи). Старт.

### 7.7. Турниры (`/tournaments`, `/tournaments/{id}`)
- Список турниров (карточки: участники, прогресс «сыграно/всего», статус).
- Детали турнира: **таблица результатов** (ранг, модель, партии, W-Н-П, очки,
  score %, точность), **расписание** партий со ссылками на каждую. Для идущего —
  авто-обновление прогресса. 🆕 устойчивость к перезагрузке.

### 7.8. Аккаунт 🆕 (`/login`, `/register`, `/profile`, `/my`)
- См. §6: формы входа/регистрации, профиль с вкладками, «Мои партии».

## 8. Система оценок ходов (★-анализ) — палитра

Каждый ход помечается знаком и **цветом** (в списке ходов и бейджем в панели хода):

| Знак | Класс | Смысл | Цвет |
|------|-------|-------|------|
| `!!` | brilliant | блестящий ход / жертва, сохраняющая перевес | **синий** |
| `!`  | good | сильный/лучший ход | зелёный |
| `!?` | interesting | острый/спекулятивный ход, требует проверки | бирюзовый |
| `?!` | inaccuracy | сомнительный ход (неточность) | янтарный |
| `?`  | mistake | ошибка | оранжевый |
| `??` | blunder | грубейшая ошибка (зевок) | красный |
| —    | normal | просто ход без улучшения/ухудшения | нейтральный |

Дизайн должен заложить эти 7 состояний в обеих темах (бейджи + цвет текста хода).

## 9. Библиотека компонентов (для дизайн-системы)

- **Кнопки:** primary (сукно), secondary (контур), icon-button (тема/язык), опасные
  (пауза/стоп — нейтральные, не «красные» без нужды).
- **Карточки/панели:** партия, турнир, модель, профиль; заголовок-метка, тело.
- **Доска:** рамка с координатами, тёплые клетки, зелёная подсветка хода, слой
  анимации фигуры. (Светлая/тёмная рамка под тему, сама доска — тёплая.)
- **Список ходов:** строки-пары (номер · белый · чёрный), глиф+цвет оценки,
  подсветка текущего, скролл.
- **Бейджи:** статус партии (идёт/пауза/завершена/ошибка), режим модели
  (think/flash), классификация хода (7 цветов).
- **Формы:** селект модели, чекбоксы, поля логина/регистрации, тоглы (тема, язык,
  стратегия, подсказки).
- **Таблицы:** standings (табличные цифры, зебра, сортировка).
- **Слайдер/контролы плеера:** ◀ ▶, ползунок (зелёный), счётчик «N / M».
- **Навигация:** header-nav с активным пунктом; меню пользователя (дропдаун);
  переключатели темы и языка.
- **Модалки/тосты:** выбор языка при входе, подтверждения, ошибки, уведомления.
- **Состояния данных:** загрузка (скелетон), пусто, ошибка — для каждого списка/экрана.

## 10. Сводный список функций

**Существующие:**
- Партия двух LLM (OpenAI/Anthropic/Gemini), валидация ходов, лог, PGN.
- Живой просмотр по WebSocket: доска, ходы парами, мысли модели, анимация
  скольжения (~chess.com), зелёная подсветка последнего хода.
- ★-анализ: оценки ходов (7 классов, цветные), точность и счётчики по сторонам,
  ключевые моменты, подсказки Stockfish (3/партию), приватная «стратегия» модели.
- Интерактивный отчёт + скачиваемый offline-HTML, экспорт PGN.
- Турниры round-robin: создание, список, таблица результатов, расписание.
- Доступ «по секретной ссылке» (токен).
- Светлая/тёмная тема (рестайл) 🆕 как часть нового дизайна.

**Новые (заложить в дизайн):**
1. 🆕 Язык интерфейса **только EN или только RU** (выбор при входе + переключатель),
   язык ответов моделей следует выбору.
2. 🆕 **Больше моделей** + выбор режима **«думающая» / «flash»** для каждой стороны.
3. 🆕 **Регистрация/вход**, профиль, «Мои партии», опц. свои API-ключи.
4. 🆕 **Пауза и продолжение** партии (control на live-странице, статус «на паузе»).
5. 🆕 **Навигация** по сайту и **устойчивость к перезагрузке** (deep-link URL,
   восстановление состояния live-партии и турнира, без «смерти» страницы).

## 11. Что нарисовать (минимальный комплект экранов)

В обеих темах (светлая/тёмная), на десктоп и мобайл:
1. Выбор языка (вход) · 2. Главная · 3. Регистрация · 4. Вход · 5. Профиль /
«Мои партии» · 6. Новая партия (с выбором модели и режима think/flash) ·
7. Живая партия (идёт / на паузе) · 8. Отчёт о партии (с раскрашенными оценками) ·
9. Архив партий · 10. Новый турнир · 11. Список турниров · 12. Детали турнира
(таблица + расписание) · 13. Состояния: загрузка / пусто / ошибка · 14. Компоненты
(кнопки, бейджи, карточки, доска, список ходов, формы).
