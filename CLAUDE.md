## Что это за проект

**LLM Chess Arena** — арена, где две LLM-модели (OpenAI, Anthropic, Google Gemini)
играют друг против друга в обычные шахматы без контроля времени. Система проверяет
легальность каждого хода, ведёт полный лог (ходы, сообщения, рассуждения моделей,
ошибки), на выходе даёт PGN (для lichess/chess.com) и self-contained HTML-отчёт.

★-фичи: 3 подсказки на партию от Stockfish каждой модели и автоматический пост-анализ
качества ходов (блестящий / хороший / неточность / ошибка / зевок).

## Стек (кратко)

- **Python 3.11+**
- `python-chess` — валидация ходов, FEN, PGN, рендер доски (SVG)
- SDK провайдеров: `openai`, `anthropic`, `google-genai`
- `Stockfish` (UCI) — подсказки и пост-анализ
- `FastAPI` + `Uvicorn` + WebSocket — веб-UI и живой просмотр партии
- `pydantic` / `pydantic-settings` — конфиг и валидация
- `Jinja2` — HTML-отчёт
- `pytest` — тесты

## Ритуал сессии (важно для преемственности между сессиями)

Память между сессиями НЕ сохраняется — состояние реконструируется из файлов и git.
Поэтому строго соблюдай ритуал.

**В начале сессии (восстановление контекста):**
1. Прочитать `CLAUDE.md` (этот файл).
2. Прочитать `docs/PROGRESS.md` — текущая фаза, последняя/следующая задача.
3. `git log --oneline -15` — что менялось в последних сессиях.
4. Свериться с отметками `[x]` в `docs/TODO.md`.
5. Кратко подтвердить пользователю «мы здесь, делаю задачу X».

**Во время сессии:**
- Брать ровно **одну** задачу из `docs/TODO.md`. Не смешивать задачи.

**В конце сессии (фиксация состояния):**
1. Отметить `[x]` выполненную задачу в `docs/TODO.md`.
2. Обновить `docs/PROGRESS.md`: текущее состояние + строка в журнале сессий.
3. Сделать коммит `type(scope): summary` (одна задача = один коммит).
4. Если менялись решения — дописать в `docs/DECISIONS.md`.

## Правила работы в репозитории

- **Секреты только в `.env`.** Никогда не коммить ключи, не писать их в логи,
  `game.json`, PGN или отчёт. `.env` и `games/` — в `.gitignore`.
- **Источник истины по партии — `game.json`.** PGN и HTML генерируются из него.
- **Валидация ходов — через `python-chess`**, не вручную.
- **Каждая задача из `docs/TODO.md` = один коммит.** Не смешивай задачи.
- **Тесты обязательны** для `core`, `prompts`, `storage`, парсинга ходов и PGN.
  Тесты провайдеров и движка — на моках/со `skip`, если нет ключа/бинарника.
- Код пиши в стиле окружающего кода; новые модули — с короткими docstring.
