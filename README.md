# LLM Chess Arena

Арена, где две LLM-модели (OpenAI, Anthropic, Google Gemini) играют друг против
друга в шахматы. Система судит легальность ходов, ведёт полный лог и на выходе даёт
PGN (для lichess/chess.com) и HTML-отчёт.

Стадия: **проектирование** (документация готова, код ещё не написан).

---

## 📂 Какой файл за что отвечает

| Файл | Что внутри | Когда открывать |
|------|------------|-----------------|
| **CLAUDE.md** | Точка входа: стек, структура папок, правила работы, **ритуал сессии** | Первым делом / в начале каждой сессии |
| **docs/PROGRESS.md** | 🔴 **Живое состояние:** где остановились, следующая задача, журнал сессий | Чтобы понять «что сейчас» |
| **docs/TODO.md** | Список атомарных задач с галочками `[x]` (1 задача = 1 коммит) | Чтобы взять следующую задачу |
| **docs/ROADMAP.md** | Этапы (Phase 0–7) и критерии готовности каждого | Чтобы увидеть общий план |
| **docs/PROJECT_SPEC.md** | Требования, фичи, форматы данных (`config.yaml`, `.env`, `game.json`) | Чтобы уточнить «что должно делать» |
| **docs/ARCHITECTURE.md** | Слои, модули, поток партии, протокол ответа модели | Чтобы понять «как устроено» |
| **docs/DECISIONS.md** | Принятые технические решения (D-001…) и почему именно так | Чтобы узнать «почему так сделано» |
| **README.md** | Этот файл — карта документации | Для общего обзора |

**Подсказка-навигация:**
- «Где мы сейчас?» → `docs/PROGRESS.md`
- «Что делать дальше?» → `docs/TODO.md`
- «Что вообще строим?» → `docs/PROJECT_SPEC.md` + `docs/ROADMAP.md`
- «Как это устроено / почему так?» → `docs/ARCHITECTURE.md` + `docs/DECISIONS.md`

## 🚀 Как начать новую сессию (фраза для копирования)

```
Проект LLM Chess Arena в C:\Users\dmitr\Chess_LLM_Claude.
Прочитай CLAUDE.md и docs/PROGRESS.md, сверься с git log и docs/TODO.md,
скажи где мы остановились — и сделай следующую задачу.
```

В конце сессии состояние фиксируется автоматически: отметка `[x]` в `docs/TODO.md`,
строка в `docs/PROGRESS.md` и git-коммит.

## 🔑 Главные правила

- Секреты только в `.env` (не коммитятся). Несекретное — в `config.yaml`.
- Источник истины по партии — `game.json`; PGN и HTML генерируются из него.
- Одна задача из `docs/TODO.md` = один коммит.
