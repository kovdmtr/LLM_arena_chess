# LLM Chess Arena

Арена, где две LLM-модели (OpenAI, Anthropic, Google Gemini) играют друг против
друга в шахматы. Система судит легальность каждого хода (через `python-chess`),
ведёт полный лог (ходы, рассуждения, сообщения, ошибки) и на выходе даёт PGN
(для lichess/chess.com) и self-contained HTML-отчёт с интерактивным плеером.

★-фичи: до 3 подсказок на партию от Stockfish каждой модели и автоматический
пост-анализ качества ходов (блестящий / хороший / неточность / ошибка / зевок).

Стадия: **реализовано** — игровое ядро, 3 провайдера, движок-подсказки и анализ,
артефакты (`game.json`/PGN/HTML) и веб-интерфейс готовы; идёт закалка (Phase 7).
Готовый пример вывода — в [`examples/sample-game/`](examples/sample-game/).

## 🚀 Быстрый старт

```powershell
# 1. установка (Python 3.11+); editable + dev-зависимости
pip install -e ".[dev]"

# 2. ключи провайдеров — в .env (НЕ коммитится; см. .env.example)
copy .env.example .env   # затем впишите свои ключи

# 3. тесты
.\.venv\Scripts\python.exe -m pytest

# 4. веб-интерфейс (выбор моделей, запуск партии, живой просмотр, отчёты)
.\.venv\Scripts\python.exe -m uvicorn arena.web.app:app
#   → откройте http://127.0.0.1:8000/

# 5. пример партии (game.json + game.pgn + report.html) без сети и ключей
.\.venv\Scripts\python.exe scripts\generate_sample_game.py
```

- **Несекретные настройки** (модели, движок, лимиты, ретраи) — в `config.yaml`.
- **Секреты** (API-ключи) — только в `.env`. В логи/артефакты ключи не попадают
  (маскируются, D-003/D-023).
- **Stockfish** (опционально): бинарник в `PATH` или путь в `config.yaml → engine.path`.
  Без него партии играются на базовом уровне — без подсказок и пост-анализа (D-008/D-024).

## ✨ Что умеет

- Партия двух моделей без контроля времени; легальность судит `python-chess`.
- Строгий JSON-протокол ответа модели (`reasoning`/`move`/`request_hint`/`resign`),
  ретрай нелегального хода и техническое поражение после 3 попыток (D-006).
- ★ Подсказки Stockfish (≤3 на сторону) и ★ пост-анализ с классификацией ходов и
  сводкой точности (D-009/D-010).
- Артефакты из единственного источника истины `game.json`: `game.pgn` и
  self-contained `report.html` (инлайн-доски, интерактивный плеер).
- Веб-UI на FastAPI: выбор моделей, запуск, живой просмотр по WebSocket, список и
  отчёты завершённых партий.
- Закалка: ретраи провайдеров с backoff (D-022), структурное логирование с
  маскированием секретов (D-023), единый путь деградации без движка (D-024).

## 🔑 Главные правила

- Секреты только в `.env` (не коммитятся). Несекретное — в `config.yaml`.
- Источник истины по партии — `game.json`; PGN и HTML генерируются из него.
- Одна задача из `docs/TODO.md` = один коммит.
- Каждая задача с кодом — с тестами и зелёным `pytest` (см. CLAUDE.md, DoD).
